◆ make_orthographic_projection()

mat4 dyx::make_orthographic_projection ( float width,
float height )
inline

Builds an orthographic projection for 2D rendering.

Maps pixel coordinates to normalized device coordinates, with the origin (0, 0) in the top-left corner of the screen and the y axis pointing down (like most 2D drawing APIs).

Note on conventions: the engine follows glm, so matrices are stored column-major (m[column][row]) and positions are column vectors that are transformed as v' = M * v. The translation lives in the last column of the matrix.

Parameters
widthWidth of the render area in pixels.
heightHeight of the render area in pixels.
Returns
Orthographic projection matrix.

Builds the world transformation for a sprite quad.

The renderer draws a unit quad from (-0.5, -0.5) to (0.5, 0.5). This transform scales it to the requested size, rotates it around its center and then moves it so its center sits at the given position.

◆ enumerate_and_score_adapters()

std::vector< adapter_info > dyx::enumerate_and_score_adapters ( IDXGIFactory1 * factory)

Enumerates all directx 12 capable adapters on this machine and scores them.

Similar to how unreal picks its gpu: every adapter gets a score based on its dedicated video memory, maximum feature level and highest supported shader model. Software adapters (warp) are listed but score zero, so they are only picked when no real graphics card is present.

Parameters
factoryFactory used to enumerate the adapters.
Returns
All directx 12 capable adapters, in enumeration order.
