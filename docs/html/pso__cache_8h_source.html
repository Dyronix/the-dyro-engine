13 /// @brief Caches pipeline state objects (psos) on disk between runs.
14 ///
15 /// Creating a pso makes the driver compile our shader bytecode into
16 /// real gpu instructions, which is slow. Directx 12 offers the
17 /// ID3D12PipelineLibrary to avoid paying that cost every run: created
18 /// pipelines are stored in the library, the library is saved to disk on
19 /// shutdown and loaded again on the next start.
20 ///
21 /// The cache invalidates itself automatically: when the driver, the gpu
22 /// or a shader changed, loading fails and the pipeline is simply created
23 /// from scratch (and stored again).
24 class pso_cache
25 {
26 public:
27 //----------------------------------------------------------
28 /// @brief Loads the pipeline library from disk (or starts an empty one).
29 /// @param d3d_device Device the pipelines are created on.
30 /// @param cache_file File the pipeline library is stored in.
31 /// @return True when the cache is ready for use (an empty cache is also fine).
32 bool initialize(ID3D12Device* d3d_device, const std::filesystem::path& cache_file);
33
34 //----------------------------------------------------------
