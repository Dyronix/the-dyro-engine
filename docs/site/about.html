About BuzEngine

A 2D game engine written for a first-year university programming course.

BuzEngine is a 2D game engine built on DirectX 12. It was written for the first-year Programming course at Breda University of Applied Sciences (BUas), where students use it to build small games in C++.

Its purpose is to support those projects. The engine handles the low-level systems such as windowing, rendering, input and asset loading, so students can spend their time on the game itself and on learning C++, rather than on the plumbing an engine needs before anything appears on screen.

Who it is for

The primary audience is first-year students who are learning C++ and working with a real-time renderer for the first time. It is also a usable starting point for anyone building a small 2D game on Windows.

The engine ships with two example games: simple_game, a "catch the ball" starter, and buz_game, a demo that exercises more of the engine. Start from either and replace it with your own.

What you'll practise

Building a game on BuzEngine means writing object-oriented C++ and putting the core ideas of 2D game programming into practice: drawing sprites, animating sprite sheets, reading keyboard and mouse input, checking collisions with rectangles, and moving a camera through a world larger than the screen. The aim is to finish able to build a complete 2D game.

How your game fits in

Your game is a class that derives from buz::game and overrides four functions: initialize, update, draw and shutdown. The engine owns the main loop and calls them at the appropriate times.

The engine itself is a set of separate classes: the window, the device, the swap chain, the 2D renderer, input, and texture and font loading. Each is documented in the API reference.

At a glance

  • Language: C++20, using std::format, concepts, ranges and std::span.
  • Graphics: DirectX 12, with no cross-platform abstraction layer.
  • Platform: Windows 10 and 11, 64-bit only. No macOS or Linux build.
  • Toolchain: Visual Studio 2022 (17.5+) or 2026 with the C++ desktop workload, and CMake on your PATH.
  • License: MIT.

BuzEngine is a 2D engine only. It does not include 3D rendering, a scene editor, a scripting layer or cross-platform builds.

How the documentation is put together

The site you are reading has two clearly separated halves. The narrative half (this page, the home page and the guides) is written by hand in Markdown and rendered into these pages by a tiny script. The code half (the API reference) is generated with Doxygen straight from the engine headers, so it never drifts from the code. The guides cross-link into that reference, so a mention of renderer_2d is one click from its full API.
